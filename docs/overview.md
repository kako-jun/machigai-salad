# 概要

## きっかけ

レストランのランチョンマットには、長年にわたって愛されている間違い探しが掲載されていることがある。
子どもたちは夢中になるが、2枚の絵を目で行き来しながら違いを探すのは意外と疲れる。
「スマホで撮って、タップひとつで切り替えながら比較できたら楽しいのに」という素朴な願いがこのアプリの出発点だ。

## 何をするアプリか

撮影した間違い探しの画像を、左右2枚に自動分割して比較できるようにするブラウザアプリ。
画面をタップしている間だけ2枚目が表示され、離すと1枚目に戻る。
この「押している間だけ切り替わる」操作が、目で差分を追いかけるよりずっと直感的で楽しい。

## 設計の柱：完全クライアント処理

画像は一切外部に送信されない。撮影した写真がサーバーに渡ることはなく、すべてブラウザの中だけで完結する。
これは単なる実装の都合ではなく、意図的な選択だ。

- 家族の食事中に撮った写真をアップロードしたくない
- 子どもが使うアプリだからこそ、プライバシーに慎重でありたい
- サーバーコストをゼロにすることで、長く無料で提供し続けられる

外部APIも従量課金のAIサービスも使わない。ブラウザだけで動く画像処理ライブラリを活用することで、この3つを同時に実現している。

## ターゲット

主なユーザーは外食中の家族連れ、特に小学生の子どもを連れた家庭を想定している。
難しい操作は不要で、「写真を撮る→ 画面をタップする」だけで使える。

専門知識がなくても使えることを最優先にしているため、UIの言葉は平易で、絵文字を多用している。
大人も使えるが、子どもでも迷わず操作できる水準を目標にしている。

## 無料・無広告・無ログイン

アカウント登録は不要。広告も表示しない。起動してすぐ使える。
静的サイトとしてホスティングしているため、サーバー費用がかからず、無料で提供し続けられる構造になっている。
