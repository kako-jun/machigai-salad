# 機能一覧

## カメラ撮影・画像選択

スマートフォンのカメラで直接撮影するか、ギャラリーから画像を選択できる。
撮影時は背面カメラが優先されるため、テーブルに置いたランチョンマットを上から撮りやすい。
ファイル選択にも対応しているため、すでに撮ってある写真を使うことも可能。

### 2枚モード

「アルバムから えらぶ」をタップすると「1まい（左右がうつってる）」と「2まい（べつべつの画像）」の選択ポップアップが表示される。

- **1まいモード（従来）**: 左右が1枚に写っている画像を読み込み → 角検出 → 台形補正 → 左右分割 → 比較
- **2まいモード**: 1枚目を選択すると「2まい目をえらぶ」モーダルが表示される。ユーザーがボタンを押して2枚目を選択し、それぞれ角検出 → 台形補正（分割なし）を行い、比較画面へ

2まいモードではABテスト、錯視画像、スクリーンショット比較など「既に別々の画像が2枚ある」ユースケースに対応する。各画像は角調整・台形補正を経るため、写真として撮った画像でも正確に比較できる。

### LINE内蔵ブラウザ対応

LINE内蔵ブラウザではカメラ権限ダイアログが表示されず、ファイル選択にフォールバックする。
このアプリはUA文字列（`Line/`）で LINE 内蔵ブラウザを検出し、OSごとに挙動を出し分ける（`lib/detectLineInApp.ts`）。
UIは Android/iOS で統一し、カメラボタンの上に「外部ブラウザで開く」誘導バナーを表示する。

- **Android**: ボタン押下で `?openExternalBrowser=1` を URL に付与して遷移。LINE 側が Chrome 等の外部ブラウザでリンクを開く
- **iOS**: iOS LINE には JS から外部ブラウザを起動する公式手段がないため、ボタン押下で案内モーダルを表示する。モーダルには「右下の『・・・』→『Safariで開く』」の手順と「URLをコピー」ボタンを用意している
- **それ以外（デスクトップLINE等）**: バナーは出さない

## 紙の自動検出・台形補正

撮影した画像から、間違い探しが印刷された紙の輪郭を自動的に検出する。
斜めから撮影した場合でも、4つの角を検出して正面から見た形に変換する（台形補正）。

検出には複数の輪郭候補を集めて外接矩形を求める方式を採用しており、
間違い探しの左右の絵が別々に検出された場合でも全体を推測できる。
片側のみ検出された場合は幅を2倍に拡張して紙全体を推測する。

## 角の手動調整

自動検出した4つの角をタッチ操作で微調整できる。
ドラッグ中の角は光るエフェクトで視覚的にわかりやすく、指の位置を見失いにくい。
調整に自信がない場合はスキップも可能で、その場合は元の画像のまま処理を続ける。

### 検出感度の切り替え

角調整画面に「もう1回けんしゅつ」ボタンがあり、検出感度を3段階（きびしめ→ふつう→あまめ）で切り替えて再検出できる。
全レベルで外接矩形戦略を使用し、Cannyしきい値・受け入れ頂点数・面積閾値・ブラーサイズが異なる。
検出に失敗した場合はデフォルト矩形（10%マージン）にフォールバックする。

### ルーペ（拡大鏡）

角をドラッグしている間、対角の位置に3倍ズームのルーペが表示される。
指で隠れてしまう角の正確な位置を、離れた場所で確認しながら操作できる。
十字線がルーペの中心にあり、常にコーナーの位置を示す。

### 中央分割線の表示

角調整キャンバスに白い破線で中央分割線が表示される。
4つの角の位置に応じてリアルタイムに動き、左右分割の境界がどこになるかを事前に確認できる。

## 左右分割

台形補正後の画像を中央で2枚に分割する。
間違い探しは通常、同じ絵が左右に並んでいるフォーマットのため、このシンプルな分割が有効に機能する。

## タップ切り替え比較

分割した2枚の画像を重ねて表示し、タップしている間だけ2枚目に切り替わる仕組み。
指を離すと1枚目に戻る。

右画像は常に背面に配置され、左画像がその上に重なっている。
タップ（静止）すると左画像が透明になり、右画像が見える。
この操作は直感的で子どもにも説明不要。「押している間だけ変わる」という分かりやすさが特徴。

## スライド微調整

比較画面で左画像をスライドすると、位置を微調整できる（±40px）。
スライド中は左画像が半透明になり、右画像が透けて見える。
これにより、四隅指定に戻らなくても左右のずれを補正できる。

タップ（静止）とスライド（移動）は自動的に区別される（25px以上の移動でスライドと判定）。

## メッシュワープ（5点変形: 4隅+中心）

比較画面で左画像の四隅と中心にドラッグ可能なハンドルが表示される。
ハンドルをドラッグすると、4三角形アフィン変換によるメッシュワープがcanvasでリアルタイム描画され、左画像を右画像に合わせて歪ませることができる。
ドラッグ中は左画像が半透明になり、右画像との重なり具合を確認しながら調整できる。

四隅のハンドルで台形歪みや回転ずれに対応し、中心のハンドルでレンズの樽型歪み（四隅が合っていても中心がずれる現象）に対応する。
画像を4つの三角形（各辺と中心を結ぶ）に分割し、それぞれにアフィン変換を適用することで非線形変形を実現している。

## マウス・タッチ両対応

スマートフォン（タッチ操作）でもPC（マウス操作）でも同じ体験が得られる。
PointerEventsで統一されており、どちらで使っても機能が変わらない。

## ビューポート制約・高DPI対応

比較画面の画像は幅・高さともビューポート内に収まるよう制約される。
縦長画像でスクロールが必要になったり、横長画像が小さすぎたりする問題を防ぐ。

角調整のCanvas描画はdevicePixelRatioに対応しており、
Retinaディスプレイでもルーペや角ハンドルがシャープに表示される。

極端に小さい画像は角調整・比較画面ともに最低280pxに拡大表示される。

## エラー時の継続動作

紙の輪郭検出に失敗した場合でも、処理を中断しない。
角調整ステップをスキップして、元画像のまま分割・比較ステップに進む。
「検出できなかったから使えない」ではなく、「できる範囲で続ける」設計になっている。

## PWA（ホーム画面インストール）

対応ブラウザでは画面下部にインストールバナーが表示される（`beforeinstallprompt`イベント）。
「インストール」をタップするとホーム画面に追加され、アドレスバーなしのスタンドアロンモードで起動する。
「あとで」を選ぶとそのセッション中はバナーが再表示されない（sessionStorage）。
ブラウザ側のUIから直接インストールした場合も、`appinstalled`イベントでバナーを自動非表示にする。
Service Workerによるキャッシュで、2回目以降の起動が高速になる。

## 訪問カウンター

フッタ下部にNostalgicカウンター（machigai-salad-0b39a12c）を表示。
訪問回数が自動的にカウントされ、1日1回の重複防止付き。
右側にはビルド日（v2026-03-28形式）を表示。

## 画像の自動縮小

カメラやアルバムから取得した画像を、表示に必要な解像度に自動でリサイズする。
端末の画面長辺 × devicePixelRatio を上限（最大2400px）とし、超過分をcanvasで縮小してからOpenCV処理に渡す。
メモリ消費・処理時間・保存データサイズをすべて改善する。

## アンドゥ（一手戻し）

角調整画面と比較画面の両方に「もどす」ボタンがある。
ドラッグ操作や再検出の前にスナップショットを履歴スタック（最大50件）に保存し、ボタンを押すと直前の状態に戻せる。
比較画面では位置オフセット、コーナーワープ、中心ワープの全てが一括で戻る。

## SNSシェア

フッタのQRコード下に、X（Twitter）/ LINE / Web Share API（またはクリップボードコピー）のシェアボタンが並ぶ。
Web Share対応端末ではOSのシェアシートが開き、非対応ではURLコピーにフォールバックする。

## 比較結果のシェア・ダウンロード（GIF / APNG）

比較画面の「アニメーションをつくる」ボタンで、左右の画像が1秒間隔で無限ループするアニメーションを生成する。

生成後はプレビューモーダルが表示され、アニメーションを確認してから「シェアする」「ダウンロード」を選べる。

- **シェア（GIF形式）**: gif.js（Web Worker）で生成。最大480px。256色だがディザリングを無効化し、黒い点々のアーティファクトを抑えている。GIF形式なのでAndroidギャラリー、LINE、X、iOS等どこでもアニメーション再生される。Web Share非対応環境ではGIFダウンロードにフォールバック
- **ダウンロード（PNG形式）**: upng-jsでアニメーションPNGを生成。最大800px。フルカラー（24bit）の可逆圧縮で高画質。拡張子は.pngで、対応ブラウザ（Chrome, Firefox, Safari, Edge）でアニメーション表示される

フレーム描画ロジック（座標計算・mesh warp）はGIF/PNGで共通化されており、スライド微調整（offset）、四隅ワープ（cornerOffsets）、中心ワープ（centerOffset）が反映される。
フレーム1は右画像ベースに左画像を調整済みの状態で重ねて描画し、フレーム2は右画像のみ。
ユーザーが比較画面で見ている状態がそのままアニメーションに再現される。

## QRコード共有

フッタにQRコードが表示されている。
スマートフォンのカメラで読み取るだけでこのアプリを開ける。
友達や家族にその場でシェアするのに便利。

## ヘッダバナー

ヘッダにルネサンス絵画風の横長バナーが表示されている。
アプリの雰囲気づくりとブランディングを担う装飾要素。
