# 対応環境

## 動作環境の基本方針

このアプリはすべての処理をブラウザ内で完結させている。
サーバーへのリクエストはなく、静的サイトとして配信されるため、
ブラウザが動く環境ならどこでも動作する。

---

## 推奨環境

### スマートフォン（最適）

このアプリを最も自然に使える環境。

| ブラウザ             | 動作確認 |
| -------------------- | -------- |
| iOS Safari（iPhone） | 対応     |
| Android Chrome       | 対応     |

スマートフォンではカメラ直撮りができるため、テーブルに置いたまま撮影してそのまま比較できる。
縦向き・横向きどちらにも対応している。

### デスクトップ

すでに撮影した画像ファイルを読み込んで使う用途に向いている。

| ブラウザ        | 動作確認 |
| --------------- | -------- |
| Chrome          | 対応     |
| Firefox         | 対応     |
| Edge            | 対応     |
| Safari（macOS） | 対応     |

---

## カメラアクセスの要件

カメラ機能を使うにはHTTPS接続が必要。
GitHub Pages および Cloudflare Pages はどちらもHTTPSで配信されるため、問題ない。

ローカル開発環境（localhost）では、ブラウザがHTTPでもカメラを許可する場合がある。

---

## 制約・既知の制限

### OpenCV.js の初回ロード

画像処理にはOpenCV.js（CDNから読み込む）を使用している。
初回アクセス時は数MB のライブラリのダウンロードが発生するため、
ネットワーク速度によっては「じゅんび中...」の時間が数秒かかる。

2回目以降はブラウザキャッシュが効くため、待ち時間は短縮される。

### オフライン動作

現時点ではオフライン動作には対応していない。
OpenCV.js をCDNから取得するため、ネットワーク接続が必要。
将来的にPWA化・Service Worker導入によるオフライン対応を検討している。

### 画像処理の重さ

スマートフォンによっては画像処理に数秒かかる場合がある。
特に高解像度の写真（1200万画素以上など）は処理時間が長くなりやすい。
将来的な改善として、入力画像の事前リサイズを検討している。

### 縦横の制約

間違い探しが左右2枚並びのフォーマット（横長）であることを前提にしている。
縦に並んでいる場合や、3枚以上の比較には現時点では対応していない。
